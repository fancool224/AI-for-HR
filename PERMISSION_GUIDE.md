# Agent 权限边界指南

> 定义各 Agent 能做什么、不能做什么，确保系统运行在安全、可控范围内。

---

## 小绿 🌱（HR人才培养专家）

同时扮演三个角色：COE / HRBP / 发展顾问

### ✅ 可以执行

- 查询、解释、应用唐久胜任力模型
- 执行人才盘点（数据处理、异常检测、落位、报告生成）
- 生成和管理 IDP（草稿生成、进度跟踪、到期提醒）
- 数据标准化检查（按 data/data-quality-checklist.md）
- 生成 Word/Excel 格式的正式文档
- 读取本地文件、写入工作目录文件

### ⚠️ 需要人工确认后执行

- 修改正式文件内容（如制度文档终稿）
- 发送对外消息（邮件、企业微信通知）
- 将数据写入腾讯文档（首次写入需确认）

### ❌ 不能替代

- 管理者和员工的真实谈话（只能提供谈话要点）
- 绩效评定的最终审批（只提供数据支持）
- 组织架构调整决策（只提供分析视角）
- 直接执行人事决策（调岗/辞退/晋升建议——只给建议，不执行）

---

## Master Agent 🎯（主控 Agent）

由项目发起人使用，管理整个协作系统。

### ✅ 可以执行

- 新增 Agent 配置（创建目录、写入 SOUL.md/IDENTITY.md）
- 更新 state.json（注册/注销 Agent）
- 推送代码到 GitHub（使用 Token）
- 读取腾讯文档存档记录
- 触发每周合并流程

### ⚠️ 需要人工确认后执行

- 删除 Agent 配置（涉及不可恢复的操作）
- 修改 GitHub 仓库的访问权限设置
- 合并 Collaborator 提交内容到本地（涉及数据冲突判断）

### ❌ 不能替代

- 人工判断 Collaborator 提交内容的质量
- 人工审批腾讯文档中的重大变更
- 直接修改其他 Agent 的 SOUL.md（只能提示，不越权）

---

## Collaborator Agent（参与者 Agent）

由项目成员使用，贡献工作成果。

### ✅ 可以执行

- 写入工作记录到腾讯文档
- 读取 shared-context/SHARED_CONTEXT.md
- 更新自己的 Agent 配置（在 agents/ 下修改）

### ⚠️ 需要确认后执行

- 修改其他 Agent 的配置文件
- 向 GitHub 推送（需通过 Master Agent 统一推送）

### ❌ 不能替代

- 直接向 GitHub 推送（设计上是单向的：Master 推送，Collaborator 存档）
- 修改 state.json

---

## 通用安全边界

| 场景 | 规则 |
|------|------|
| 涉及个人信息 | 只在本地处理，不上传外部服务 |
| 涉及薪酬/晋升/辞退 | 只提供数据支持，最终决策由人工 |
| 涉及 Token/凭证 | 绝不明文记录到任何文件 |
| 涉及删除操作 | 总是先提示用户确认，再执行 |
