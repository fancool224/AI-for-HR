# 试用期评估预警技能

此技能用于自动化试用期评估提醒流程，从腾讯文档读取数据、筛选即将到期的评估记录，并生成分级提醒消息。

## 使用场景

当用户提出以下类型的请求时，应使用此技能：
- "检查试用期评估到期情况"
- "生成评估提醒"
- "试用期评估预警"
- "提醒李鹏/曲雅如处理评估"
- 任何涉及定期跟踪试用期评估的需求

## 数据源配置

**腾讯文档信息：**

- 表名: `01【员工关系】试用期评估台账`

## 筛选规则

**必须满足的条件：**
1. `转正状态`（列15）为空（未进行转正）
2. 任意评估日期列（17/19/21/23/25）在 `[今天, 今天+7天]` 范围内
3. 对应的评估结果列（18/20/22/24/26）为"待评估"、"催办中"或为空

**日期解析格式：**
- 优先：`YYYY/MM/DD`（如 2026/05/13）
- 备选：`YYYY-MM-DD`（如 2026-05-13）

## 责任人分配规则

```python
def get_responsible(dept1):
    if dept1 in ('销售中心', '培训部'):
        return '李鹏'
    return '曲雅如'
```

## 执行流程

### 方法1：使用sheet.get_cell_data API（推荐，已验证可用）

1. 调用腾讯文档API获取CSV格式数据：
   ```
   mcp__tencent-docs__sheet.get_cell_data
   - file_id: DVmp5RkRXSk9ZWkVY
   - sheet_id: 000001
   - return_csv: true
   - start_col: 0
   - end_col: 26
   - start_row: 0
   - end_row: 300  （分批获取，每批300行）
   ```
   注意：表格约1136行，需分批获取（每批≤300行，单次返回≤80000字符）

2. 解析CSV数据，应用筛选规则

3. 输出格式化提醒消息

**⚠️ API避坑说明：**
- `mcp__tencent-docs__smartsheet.list_records` 不可用（报错 SheetEngine Service Error）
- `mcp__企业微信文档__smartsheet_get_records` docid格式不匹配（该文件是腾讯文档非企业微信文档）
- `mcp__tencent-docs__sheet.get_cell_data` 是唯一验证可用的API

### 方法2：使用Python脚本

直接执行 `scripts/filter_eval.py` 脚本，该脚本会：
- 读取CSV数据（从API下载或本地文件）
- 应用筛选规则
- 生成提醒消息

## 自动化配置

此技能支持通过 WorkBuddy 自动化功能定期执行。

**建议的自动化配置：**
- **名称**：试用期评估每周提醒
- **_scheduleType_**：recurring
- **_rrule_**：`FREQ=WEEKLY;BYDAY=MO;BYHOUR=10;BYMINUTE=0`
- **_prompt_**：检查本周试用期评估到期情况，生成提醒消息
- **_cwds_**：`C:\Users\Administrator\WorkBuddy\automation-claw-2026-05-09-task-1`

## 注意事项

1. **API授权**：确保腾讯文档MCP服务已连接且授权有效
2. **日期格式**：处理多种日期格式，避免格式错误导致漏检
3. **空值处理**：评估结果可能为空，应按"待评估"处理
4. **责任人确认**：输出前确认责任人分配是否正确
5. **消息送达**：提醒消息应通过企业微信等方式送达责任人
