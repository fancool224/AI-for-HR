## 关键筛选逻辑

### 条件1：未转正（仍在试用期）
```python
# 转正状态（列15）为空
row[15].strip() == ''
```

### 条件2：评估日期在未来7天内
```python
# 任意评估日期列(17/19/21/23/25)在 [今天, 今天+7天] 范围内
eval_date >= today AND eval_date <= today + 7
```

### 条件3：评估未完成
```python
# 对应评估结果列(18/20/22/24/26)为"待评估"、"催办中"或为空
row[result_col].strip() in ('', '待评估', '催办中')
```

## 日期格式处理

台账中的日期可能存在多种格式，需兼容处理：

```python
def parse_date(date_str):
    """解析日期字符串"""
    if not date_str or not date_str.strip():
        return None

    date_str = date_str.strip()
    formats = [
        '%Y/%m/%d',        # 2026/05/13 (常用)
        '%Y-%m-%d',        # 2026-05-13
        '%Y年%m月%d日',    # 2026年05月13日
    ]

    for fmt in formats:
        try:
            return datetime.strptime(date_str, fmt)
        except ValueError:
            continue
    return None
```

## ⚠️ 历史Bug记录

### 列索引错误（已修复）
- **错误**：转正状态使用 col16，实际应为 col15
- **原因**：表头中 col15 = "转正状态"，col16 是空占位列。最初分析数据时误将 col16 当作转正状态
- **影响**：所有行的 col16 均为空，导致条件1（转正状态为空）永远为真，已转正/已流失员工不会被过滤掉
- **修复**：将转正状态列从 col16 改为 col15
