# LLM Prompt：流失原因主题聚类

## 使用时机

`run_analysis.py` 跑完后，Agent 读取 `summary.json` 中的 `text_analysis.llm_input_records` 字段，按此提示词完成主题聚类。**每月强制执行一次**。

## 输入

`llm_input_records` 是数组，每条包含：
- 姓名 / 司龄段 / 二级部门 / 岗位 / 流失类别
- 面谈摘要（最多500字）

## 提示词（直接复用）

```
你是 HR 流失分析专家。下面是本月所有离职员工的"员工本人离职面谈"文本。

请完成以下任务：

1. 把这些文本聚类为 3-6 个"隐性主题"。要求：
   - 主题名要具体，避免笼统（❌"工作问题" → ✅"店长管理风格冲突"）
   - 主题应反映"机制/制度/管理"层面的可改善因素，而非员工个人因素
   - 主题之间应互斥

2. 对每个主题输出：
   - 主题名称
   - 涉及人数
   - 涉及的部门/岗位分布
   - 3-5 句代表性原话摘录（带姓名）
   - 与"流失原因分类"字段的差异分析（员工自报的浅层原因 vs 你识别的深层主题）

3. 最后输出 1-2 条"本月新出现的、过去未识别的主题"（如果有）。

输出格式（Markdown 表格 + 列表）：

## 本月隐性主题聚类

### 主题 1：{主题名}（{N} 人）
- **涉及部门**：…
- **涉及岗位**：…
- **代表原话**：
  - "…"（张三，销售主管）
  - "…"（李四，副店长）
- **深层 vs 表层**：员工自报为"职业发展"，但实际深层是…

### 主题 2：…

…

## ⚠️ 本月新出现的主题
- {主题}：…
```

## 输出去向

Agent 把 LLM 返回的 Markdown 追加到 `text_analysis.md` 末尾，或写入独立的 `topic_clustering.md`。

## Token 节约规则

- 如果 `llm_input_records` 长度 > 80 条，分批调用（每批 60 条），最后再合并主题
- 单条 `面谈摘要` 已截断到 500 字
- 优先用 GPT-4o-mini / Claude Haiku 等小模型即可，不需要顶级模型
