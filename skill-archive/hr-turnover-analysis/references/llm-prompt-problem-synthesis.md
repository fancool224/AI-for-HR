# LLM Prompt：收束 3 条核心问题

## 使用时机

`run_analysis.py` 跑完、主题聚类完成后，Agent 读取 `summary.json` + `topic_clustering.md`，按此提示词收束最多 3 条核心问题。

## 提示词（直接复用）

```
你是 HR 经营分析专家。基于下方的本月流失分析数据，按 A3 问题解决法的"明确问题"环节，
**严格收束出 1-3 条核心问题**（不能多于 3 条）。

## 输入数据

### 切片信号（按信号强度排序）
{粘贴 summary.json 的 slices_signal 前 10 条}

### 三阶交叉验证
- 一致型：{N} 人
- 矛盾型：{N} 人（详见 conflict_rows）
- 缺失型：{N} 人

### 风险预警
{粘贴 summary.json 的 risks}

### 隐性主题聚类
{粘贴 topic_clustering.md}

## 筛选规则（严格执行）

按下列优先级排序，取 Top 3：

1. **风险预警里的 🔴 红色** 一定要进入候选
2. **信号强度 ⭐⭐⭐⭐ 的切片** 一定要进入候选
3. **三阶矛盾型清单中反复出现的部门/岗位/上级** 一定要进入候选
4. **主题聚类中的"本月新出现主题"** 一定要进入候选

候选 > 3 条时，按"涉及人数 × 信号强度"综合排序，取前 3。

## 输出格式（Markdown）

## 本月核心问题（最多3条）

### 问题 1：【{标签}】{现象描述}

- **数据支撑**：…（精确人数+占比+同比）
- **影响范围**：涉及 X 个部门 / Y 个岗位 / Z 名员工
- **三阶证据**：…（引用 1-2 个具体姓名+口径）
- **建议责任人**：…（基于二级部门归属推断）

### 问题 2：…

### 问题 3：…

## 不进入 Top3 的理由说明

简短说明哪些候选问题被淘汰、为什么。
```

## 输出去向

Agent 把 LLM 返回的内容写入 `core_problems.md`。

## 后续衔接

收束完 3 条问题后，Agent 应按 `references/intervention-roadmap.md` 为每条问题输出"真因 + 策略 + 执行计划"。
