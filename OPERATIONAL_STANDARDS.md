## 一、云文档连接器规范

### 腾讯文档（docs.qq.com）

**触发条件**：用户要求在腾讯文档空间创建/编辑文件时，**必须使用腾讯文档MCP连接器**，不得手动生成本地文件替代。

**配置方法**：
1. 用户访问 https://docs.qq.com/open/auth/mcp.html 获取个人 Token
2. 在 `~/.workbuddy/mcp.json` 中添加配置：

```json
{
  "mcpServers": {
    "tencent-docs": {
      "url": "https://docs.qq.com/openapi/mcp",
      "headers": {
        "Authorization": "用户Token值"
      }
    }
  }
}
```

**Header 格式要求**：
- Header key **必须**为 `Authorization`（不得用 token / auth / X-Token 等）
- Token 值必须从官方链接获取，不得硬编码

**文件类型对应规则**：
| 文件类型 | 腾讯文档产品 |
|---------|------------|
| 数据汇总、记录表、进展追踪 | → 新建「智能表格」 |
| 说明文档、使用手册、共享历史 | → 新建「文档」 |

**权限限制**：
- 普通用户：100次/天
- 超级会员：1000次/天
- 超级会员Plus：2000次/天

**常见错误处理**：
| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 400006 | Token鉴权失败 | 检查Header key是否用Authorization，Token是否正确，重新获取 |
| 400007 | VIP权限不足 | 提示用户购买超级会员 |

## 二、本地文件 vs 云端协作规范

| 场景 | 存储位置 |
|------|---------|
| Agent配置文件、源代码 | 本地 + GitHub |
| 私有Token、敏感配置 | 本地 .gitignore 保护 |
| 协作成果存档、共享记录 | 腾讯文档 |
| 共享上下文历史记录 | 腾讯文档 |

> **重要**：涉及腾讯文档的操作，必须通过 MCP 连接器完成，不能只在本地生成 Markdown 文件。
